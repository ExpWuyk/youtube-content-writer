# YouTube 视频内容文案生成

## 工作流程

### 1. 提取视频元数据

运行内置脚本获取视频信息：

```bash
python "<skill-dir>/scripts/fetch_video_meta.py" "<youtube_url>"
```

输出字段：TITLE、CHANNEL、UPLOADER、UPLOAD_DATE、DURATION、VIEW_COUNT、LIKE_COUNT、DESCRIPTION、TAGS、CATEGORIES、LICENSE。

### 2. 补充搜索（必做）

用 WebSearch 搜索以下信息以补充视频内容理解：

- **歌词/台词/内容解析**：搜索 `"<视频标题>" 歌词/解析/考察/review`，获取视频正片内容细节
- **热度/传播情况**：搜索 `"<视频标题>" bilibili/抖音/viral/TikTok`，了解国内平台传播数据
- **背景信息**：搜索作者/IP/作品背景，为文案提供深度

每个维度至少 1 次搜索，可并行。目标：尽可能还原视频正片内容，而非仅依赖 description。

### 3. 生成全套文案

读取 `references/content-prompt.md` 获取完整输出模板和规则，严格按照其中格式输出：

**输出结构：**

```
## 第一步：视频核心信息解析
### 1. 核心定位
### 2. 高光亮点
### 3. 合规信息
### 4. 关键词库

## 第二步：全套可直接复制发布文案
### 1. 作品标题（5组，均≤30字）
### 2. 作品简介（完整版/精简版/极简版）
### 3. 话题标签（20个，带#，分层级）
```

## 硬性规则

- 标题单条 ≤ 30 字，无例外
- 所有内容必须基于视频实际内容，禁止空泛套话
- 必须包含原素材来源标注（作者、频道、发布时间）
- 规避敏感词和限流风险
- 输出纯文本发布文案，禁止生成剪辑/分镜等内容
- 全部中文输出（专有名词保留原文）
